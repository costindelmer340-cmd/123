# 开发计划

本项目当前定位为抖音优先的电商售后中台。系统不自建购物链路，先接入抖音店铺，统一处理外部订单、售后、客服、工单、评价分析和 AI 辅助能力；后续可扩展淘宝、京东、拼多多，或在同一底座上补充自建商城模块。

## 阶段一：项目重定位

- 将项目定位从完整购物平台调整为售后客服与评价分析中台。
- 明确抖音店铺绑定、订单同步、售后同步、评价同步和售后回写是首期重点。
- 调整 README、开发计划、接口规范、数据库设计说明和部署说明。
- 保留消费者端、商家客服端、平台管理员端和 AI 服务四部分架构。

## 阶段二：数据库与核心模型重构

- 新增外部平台表、抖音店铺绑定表、平台授权表。
- 新增订单同步任务表、同步日志表、外部接口调用日志表。
- 将订单、商品、支付、物流调整为外部平台快照模型。
- 保留售后申请、工单、客服会话、评价、知识库、AI 配置、操作日志等核心表。
- 更新初始化脚本、演示数据和 ER 文档。

## 阶段三：后端基础框架

- 搭建 Spring Boot 工程，包名为 `com.example.mall`。
- 集成 MyBatis-Plus、MySQL、Redis、Spring Security、JWT、Swagger/OpenAPI。
- 建立统一响应、异常处理、参数校验、分页、审计字段和基础模块结构。

## 阶段四：认证与权限

- 实现登录、注册、JWT 签发和刷新。
- 支持 `CONSUMER`、`MERCHANT_ADMIN`、`CUSTOMER_SERVICE`、`PLATFORM_ADMIN` 四类角色。
- 完成接口权限控制、商家数据隔离和店铺授权数据隔离。

## 阶段五：抖音平台接入

- 创建抖音平台配置模型。
- 实现商家绑定抖音店铺。
- 实现授权回调、token 保存、token 刷新和授权失效提醒。
- 实现抖音订单同步、订单详情同步、物流同步。
- 实现抖音售后/退款同步。
- 实现抖音评价同步。
- 实现售后处理结果回写抖音，并记录回写日志。

当前阶段五已完成基础骨架：平台配置查询、商家店铺绑定、模拟授权 Token 保存、同步任务创建、手动触发同步和同步日志查询。真实抖音开放平台 API 调用、OAuth 回调、订单/售后/评价真实数据落库和售后回写将在拿到开放平台应用凭证后继续替换模拟实现。

## 阶段六：订单与售后中台

- 实现外部订单列表、订单详情、订单状态和物流状态查看。
- 实现售后申请、资料补充、客服审核、处理记录和进度查看。
- 实现退款、退货、换货、维修等售后流程。
- 实现售后状态与外部平台状态映射。

当前阶段六已完成基础能力：商家侧外部订单列表/详情、售后列表/详情、消费者发起售后、商家审核售后、退款快照、外部售后映射和回写日志。后续会继续补充更严格的订单归属校验、售后材料上传和外部平台真实回写。

## 阶段七：实时客服与工单

- 后端使用 Spring WebSocket + STOMP。
- Web 后台和小程序使用 WebSocket 接入实时对话。
- 实现会话列表、消息记录、消息已读、转人工和 AI 辅助回复。
- 实现咨询转工单、售后转工单、工单分配、状态流转和处理日志。

当前阶段七已完成基础能力：WebSocket/STOMP 配置、消费者发起会话、消息落库和广播、AI 简单自动回复、商家会话列表、转人工、关闭会话、服务评价、工单创建、工单列表、工单详情和工单状态流转。后续前端开发阶段会把这些接口接入实际聊天窗口和工单工作台。

## 阶段八：AI 专用能力

- 使用 FastAPI 搭建 AI 服务。
- 首期使用规则和关键词实现意图识别、情感分析、主题归类、工单分类和辅助回复。
- 后续按售后语料训练专用小模型，优先覆盖意图识别、工单分类、情感分析和主题归类。
- 回复生成层预留开源模型微调和 RAG 知识库能力。

当前阶段八已完成基础能力：FastAPI AI 服务、健康检查、意图识别、情感分析、主题归类、工单分类、辅助回复接口，以及 Spring Boot 后端 `/api/merchant/ai/**` 统一调用入口。当前实现为轻量规则版，后续可替换为专用小模型推理服务。

## 阶段九：评价分析与知识库

- 同步外部平台评价，同时支持本系统服务评价。
- 实现评价情感趋势、主题分布、差评预警和高频问题统计。
- 实现 FAQ、售后规则、品牌回复口径和商品售后知识管理。

当前阶段九已完成基础能力：商家侧评价列表、评价详情、评价 AI 分析入库、知识库文章管理、FAQ 管理和售后规则管理。评价分析会调用阶段八 AI 服务生成情感、主题、关键词、风险等级和摘要。

## 阶段十：前端多端开发

- 消费者小程序：订单查询、客服咨询、售后申请、进度查看、补充资料、评价提交。
- 商家后台：店铺绑定、订单同步、售后处理、实时客服、工单、评价分析、知识库。
- 管理员后台：平台配置、用户、商家、客服、规则、AI 配置、同步监控和统计分析。

当前阶段十已完成商家客服后台首版：Vue 3 + TypeScript + Vite + Element Plus 工程、后台布局、工作台、店铺绑定、外部订单、售后处理、实时客服、工单处理、评价分析和知识库页面。当前页面先使用模拟数据展示业务形态，后续继续接入真实后端接口，并补齐消费者微信小程序和管理员后台。

本阶段进一步补齐了平台管理员后台首版界面，以及消费者微信小程序的原生页面骨架；当前商家后台、管理员后台和消费者微信小程序都已有可进入的首版页面，后续主要是接口联调与交互细化。

消费者微信小程序当前已增加登录页和演示登录入口，首页、订单、售后、客服、我的页面可以在微信开发者工具中直接打开查看。

## 阶段十一：部署与联调

- 编写 Docker Compose 和 Nginx 配置。
- 联调 MySQL、Redis、后端、AI 服务和各前端。
- 准备抖音模拟数据、演示账号、演示店铺和启动说明。

当前阶段十一已完成基础部署骨架：根目录 `docker-compose.yml`、`.env.example`、后端 Dockerfile、AI 服务 Dockerfile、商家后台和管理员后台 Dockerfile，以及统一 Nginx SPA/接口/WebSocket 代理配置。当前配置面向本地开发演示，可一键启动 MySQL、Redis、后端、AI 服务、商家后台和管理员后台；同时新增 `scripts/dev-start.ps1`、`scripts/dev-stop.ps1` 和 `scripts/dev-check.ps1`，用于本机开发模式启动、停止和联调检查。本机开发模式下，后端健康检查、Swagger、AI 服务、商家后台、管理员后台、商家登录、后端调用 AI 情感分析链路均已通过。完整 Docker Compose 容器启动仍受 Docker Engine 状态和 Docker Hub 镜像拉取影响，需要在配置代理/镜像源或导入离线镜像后继续验证。

## 阶段十二：测试与优化

- 接口测试、权限测试、订单同步测试、售后回写测试、实时客服测试、AI 结果测试。
- 优化页面体验、接口性能、异常提示、同步重试和部署稳定性。
